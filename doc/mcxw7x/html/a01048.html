Construct a key derivation function (KDF) mode according to NIST SP800-56C.

Parameters
[out]pDerivationModePointer to the key derivation mode structure to be initialized.
[in]derivationAlgorithmFunction pointer to the key derivation algorithm function.
[in]macModeMac mode type, in case a Mac mode is used within the key derivation algorithm.
[in]hashFunctionhash function type, in case a hash function is used within the key derivation algorithm. SecSha algorithm descriptors are not supported.
[in]optionsGeneric options field containing algorithm-specific parameters (e.g., counter length or iteration count). Currently not used.

When calling mcuxClKey_derivation function with this constructor additional inputs may be provided via mcuxClKey_DerivationInput_t inputs[] parameter array. Detailed description of possible inputs is provided for each algorithm type.

Returns
status
Examples
mcuxClKdfModes_SP800_56C_example.c, mcuxClKdfModes_SP800_56C_twostep_cmac_example.c, and mcuxClKdfModes_SP800_56C_twostep_hmac_example.c.

static

SP800 56C TwoStep derivation algorithm type.

When using this algorithm possible inputs[] for mcuxClKey_derivation function are:

  • inputs[0] = Label
  • inputs[1] = Context
  • inputs[2] = salt (length is required, data pointer may be NULL which defaults to all-zero salt)
